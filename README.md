# 얕넓 v0.1

얕고 넓은 지식을 위한 모바일앱 MVP입니다.

## 실행 방법

먼저 Node.js LTS를 설치한 뒤, VSCode에서 이 폴더를 열고 터미널에서 실행하세요.

```powershell
npm install
npm run start
```

Expo 개발 서버가 뜨면 휴대폰의 Expo Go 앱으로 QR 코드를 스캔해 확인할 수 있습니다.

## 현재 구현된 기능

- 홈 탭: 오늘의 지식 3개
- 탐색 탭: 카테고리/전체 지식 목록
- 복습 탭: 더미 복습 목록
- 마이 탭: 더미 학습 통계
- 지식 상세: 카드 5장 넘기기
- 퀴즈: 선택지 선택 후 정답/오답/해설 표시
- 북마크: 화면 상태 기준 토글

## 아직 연결하지 않은 것

- Supabase DB
- 로그인
- 실제 북마크 저장
- 실제 학습 기록 저장
- 푸시 알림
- 결제/구독

## 제품 방향성 메모

얕넓은 단순한 지식 콘텐츠 앱이 아니라, 사용자의 지적 호기심을 설계하는 앱을 목표로 한다.

사용자가 앱을 열었을 때 기대하는 핵심 경험은 다음 흐름이다.

1. "이게 뭐지?"라는 호기심이 생긴다.
2. "이게 내 생활이나 세상과 연결되어 있네"라고 느낀다.
3. 어려운 개념이 짧은 카드 몇 장으로 이해된다.
4. 퀴즈를 통해 "방금 내가 이해했구나"라는 작은 쾌감을 얻는다.
5. 다음 주제도 하나 더 보고 싶어진다.

Perplexity의 뉴스 카드/Discover 경험에서 받은 자극처럼, 얕넓도 세상이 갑자기 조금 더 연결되어 보이는 느낌을 주는 것이 중요하다. 다만 뉴스 기사 자체를 복제하거나 요약하는 앱이 아니라, 뉴스와 일상에서 자주 마주치는 개념을 이해 가능한 작은 지식 단위로 바꾸는 앱을 지향한다.

## 콘텐츠 설계 원칙

- 제목은 설명형보다 호기심을 유발하는 질문형/생활 연결형을 우선한다.
- 카드 본문은 전문 용어를 그대로 늘어놓기보다 일상 언어로 번역한다.
- 각 지식은 "왜 중요한가"가 먼저 느껴져야 한다.
- 마지막에는 사용자가 머릿속에 가져갈 수 있는 핵심 한 문장을 남긴다.
- 퀴즈는 시험이 아니라 이해 확인과 작은 성취감을 주는 장치로 설계한다.
- 출처가 필요한 주제는 추후 공식/신뢰 가능한 자료를 연결하되, 원문 콘텐츠를 무단 복제하지 않는다.

예시 방향:

- "금리는 왜 오르고 내릴까?"보다 "금리가 오르면 왜 내 월세까지 흔들릴까?"
- "환율이란 무엇인가?"보다 "환율이 오르면 왜 내 장바구니가 비싸질까?"
- "AI 환각 현상"보다 "AI는 왜 틀린 말을 자신 있게 할까?"

## 개발 진행 큰 흐름

앞으로의 수정과 추가는 아래 순서를 큰 틀로 따른다.

1. MVP 화면과 흐름 완성
2. 로컬 상태 기반 기능 완성
3. DB/Supabase 연결
4. 개인화, 복습, 추천 고도화
5. 출시 준비

현재는 `1. MVP 화면과 흐름 완성`에서 `2. 로컬 상태 기반 기능 완성`으로 넘어가는 단계다.

## 테스트 기준

회사 PC에서는 회사망 제한으로 Expo Go 모바일 테스트가 어려울 수 있다. 이 경우 웹에서 화면, 라우팅, 기본 UI, 상태 흐름을 확인한다.

```powershell
npm install
npm run web
```

집 PC에서는 Expo Go로 실제 휴대폰 테스트를 진행한다.

```powershell
npm install
npm run start
```

회사망에서 `exp://...` 접속이 실패하거나 인터넷 연결 오류가 나더라도, 회사 네트워크 정책 때문일 수 있으므로 앱 코드 문제로 바로 단정하지 않는다.

## 콘텐츠 수급과 저작권 원칙

얕넓은 새로운 기술, 발견, 조사결과, 뉴스 이슈를 다룰 수 있다. 다만 기사 본문이나 특정 매체의 표현을 가져와 요약 앱처럼 재배포하지 않고, 여러 신뢰 가능한 자료를 바탕으로 얕넓 고유의 설명 카드로 새로 작성한다.

기본 원칙:

- 사실, 아이디어, 개념, 연구 주제 자체는 활용할 수 있다.
- 기사 문장, 문단 구성, 독창적인 표현, 이미지, 도표를 그대로 가져오지 않는다.
- 유료 기사, 로그인 뒤 콘텐츠, 차단된 사이트, 크롤링 금지 의사가 명확한 페이지는 수집 대상으로 삼지 않는다.
- 하나의 기사만 보고 카드화하지 않고, 가능하면 공식 발표, 논문, 기관 자료, 복수 매체를 교차 확인한다.
- 최종 앱 문구는 얕넓 문체로 새로 작성한다.
- 참고한 자료는 추후 `sourceRefs` 같은 필드로 남긴다.
- 불확실하거나 최신성이 중요한 내용은 카드 안에서 단정하지 않고 기준 날짜와 출처를 함께 관리한다.

권장 흐름:

1. 주제 후보 수집: 뉴스 제목, 공식 발표, 논문, 보고서, 트렌드 키워드에서 흥미로운 주제를 찾는다.
2. 사실 확인: 원자료, 공식기관, 논문 초록, 신뢰 가능한 복수 출처로 핵심 사실을 확인한다.
3. 개념 추출: 사건 자체보다 사용자가 이해하면 좋은 원리, 배경, 연결고리를 뽑는다.
4. 얕넓식 재작성: 제목, 요약, 왜 알아야 할까, 카드 5장, 퀴즈 1개를 새 문장으로 작성한다.
5. 검수: 사실 오류, 과장, 저작권 위험, 너무 어려운 표현, 출처 누락을 확인한다.
6. 저장: 초기에는 `knowledgeData.ts`, 이후에는 Supabase 같은 DB에 저장한다.

좋은 변환 예시:

- 기사 주제: AI 반도체 수요 급증
- 얕넓 카드: "AI 시대에 왜 작은 칩 하나가 국가 경쟁력이 될까?"

- 기사 주제: 새로운 우주망원경 관측 결과 발표
- 얕넓 카드: "빛을 보면 왜 과거의 우주를 볼 수 있을까?"

- 기사 주제: 금리 동결 발표
- 얕넓 카드: "금리를 안 올리는 것도 왜 중요한 결정일까?"

이 앱은 기사 복제나 단순 뉴스 요약보다, 사용자가 세상을 이해하는 개념과 맥락을 얻는 경험을 우선한다.
